# ROADMAP

## Текущая задача: Анализ и улучшение админки "Настройки"

**Цель**: Проанализировать админку меню "Настройки", протестировать всю логику нажатий кнопок, реализовать изменения данных в настройках прямиком на сайт и сделать работоспособной

### Задачи:

- [ ] **Анализ админки меню "Настройки"**
  - URL: `http://localhost/admin/settings.php`
  - Проанализировать структуру страницы настроек
  - Изучить все доступные поля и опции
  - Зафиксировать текущее состояние настроек
  - Сделать скриншот страницы настроек

- [ ] **Тестирование логики нажатий кнопок**
  - Протестировать кнопку "Сохранить настройки"
  - Проверить кнопку "Сбросить к умолчаниям"
  - Протестировать кнопку "Предварительный просмотр"
  - Проверить кнопку "Экспорт настроек"
  - Проверить кнопку "Импорт настроек"
  - Зафиксировать результаты тестирования

- [ ] **Анализ кода страницы настроек**
  - Изучить файл `admin/settings.php`
  - Проанализировать обработку POST запросов
  - Проверить валидацию данных
  - Изучить сохранение в базу данных
  - Выявить возможные проблемы

- [ ] **Реализация изменений данных в настройках**
  - Исправить логику сохранения настроек
  - Реализовать корректное обновление данных в БД
  - Добавить валидацию полей
  - Реализовать отображение изменений на сайте
  - Добавить уведомления об успешном сохранении

- [ ] **Тестирование работоспособности**
  - Протестировать сохранение всех типов настроек
  - Проверить отображение изменений на фронтенде
  - Протестировать сброс к умолчаниям
  - Проверить экспорт/импорт настроек
  - Убедиться в корректной работе всех функций

- [ ] **Создание скриншотов для удостоверения**
  - Скриншот страницы настроек (до изменений)
  - Скриншот процесса сохранения настроек
  - Скриншот уведомлений об успешном сохранении
  - Скриншот отображения изменений на фронтенде
  - Скриншот страницы настроек (после изменений)

- [ ] **Повторная проверка (если необходимо)**
  - Если что-то не проверено - повторить тестирование
  - Если не создан ни один скриншот - сделать все скриншоты
  - Убедиться в полной работоспособности системы

## Ожидаемый результат

- Админка "Настройки" должна корректно сохранять все изменения
- Изменения должны сразу отображаться на сайте
- Все кнопки должны работать правильно
- Должны быть созданы скриншоты для подтверждения работоспособности

## Файлы для анализа

- `admin/settings.php` - основная страница настроек
- `database.php` - работа с базой данных
- `functions.php` - общие функции
- `config.php` - конфигурация системы
- `ux/layout.php` - отображение на фронтенде
- `ux/data.php` - получение данных для фронтенда

## Типы настроек для тестирования

- Основные настройки сайта (название, описание, контакты)
- SEO настройки (мета-теги, ключевые слова)
- Настройки дизайна (цвета, шрифты)
- Настройки интеграций (Google Analytics, карты)
- Настройки безопасности
- Настройки уведомлений
